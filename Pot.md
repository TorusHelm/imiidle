# Контракт Для Pot

## Назначение

Этот документ фиксирует runtime-поведение и архитектурную роль `Pot`.

Это строгий контракт для агентов, а не гайд по реализации.

## Роль Pot

`Pot` — это контейнер для `Plant`.

`Pot`:
- не является actor,
- не участвует в симуляции как самостоятельный исполнитель логики,
- не инициирует события,
- выступает как слой между `Slot` и `Plant`.

## Связь Slot -> Pot -> Plant

Правила структуры:
- `Slot` содержит либо `Pot`, либо `Totem`,
- если в `Slot` находится `Pot`, внутри него может находиться `Plant`,
- `Pot` может быть пустым,
- `Plant` не размещается напрямую в `Slot`, только через `Pot`.

## Отсутствие Активного Поведения

Жесткие правила:
- `Pot` не тикает,
- `Pot` не имеет собственного времени,
- `Pot` не генерирует события,
- `Pot` не участвует в event loop как actor.

Следствие:
- `Pot` не является источником самостоятельной активности в симуляции.

## Влияние Pot На Plant

`Pot` может влиять на `Plant`, который находится внутри него.

Это влияние:
- пассивное,
- постоянное,
- действует только пока `Plant` находится в этом `Pot`.

Примеры допустимого влияния:
- multiplier к `value`,
- изменение скорости или `cycle`,
- бонусы к offshoot,
- ограничения по типу растений.

## Модель Влияния

Влияние `Pot`:
- не является timed modifier,
- не является instant effect,
- не генерирует события каждый tick,
- не спамит event loop.

Архитектурное правило:
- `Pot` создает постоянный слой влияния, аналогичный aura,
- `Shelf` учитывает влияние `Pot` при расчете поведения `Plant`.

## Слои Расчета Поведения Plant

Итоговое поведение `Plant` может зависеть от:
- базовых параметров `Plant`,
- timed modifiers,
- aura-влияний,
- влияния `Pot`.

Это архитектурное правило расчета, а не отдельная реализация.

## Pot Не Меняет Базовые Параметры Plant

`Pot` не мутирует permanently base stats `Plant`.

Правила:
- `Pot` влияет только на итоговый расчет поведения,
- влияние `Pot` действует только пока `Plant` находится в этом `Pot`.

## Разделение Состояния

`Plant` хранит свой runtime state:
- `progress`
- `age`
- modifiers

`Pot`:
- не хранит runtime state `Plant`,
- не управляет таймингом `Plant`,
- не подменяет собой внутреннее состояние `Plant`.

## Runtime При Извлечении Plant

Если `Plant` извлекается из `Pot`:
- runtime state `Plant` сбрасывается,
- повторная посадка `Plant` начинается с нуля.

Это является частью gameplay-логики.

## Совместимость Plant И Pot

Не каждый `Plant` может быть помещен в любой `Pot`.

`Pot` может задавать ограничения:
- по типу,
- по тегам,
- по другим контейнерным условиям.

Жесткое правило:
- проверку совместимости выполняет `Shelf`,
- `Pot` не принимает решение о размещении самостоятельно.

## Targeting И Эффекты

`Pot` не является стандартной целью для modifiers и instant effects.

Правила:
- `Pot` не участвует в targeting-системе как actor,
- эффекты направляются на `Plant` или `Totem`, а не на `Pot`,
- отдельная работа с `Pot` как целью не предполагается, если это явно не зафиксировано в будущем.

## Modifiers И Pot

`Pot`:
- не хранит timed modifiers,
- не имеет списка active modifiers,
- не является частью обычной системы modifiers как цель.

Его влияние:
- является отдельным контейнерным слоем,
- не смешивается с системой timed modifiers.

## Расширяемость

`Pot` может иметь собственные параметры.

Примеры:
- качество,
- уровень,
- контейнерные бонусы и ограничения.

Правила:
- эти параметры влияют только на контейнерные свойства `Pot`,
- расширение `Pot` не превращает его в actor.

## Pot Не Должен

`Pot` не должен:
- быть actor,
- иметь собственный tick,
- иметь собственный таймер,
- генерировать события,
- напрямую взаимодействовать с другими сущностями,
- выбирать цели для эффектов,
- хранить runtime state `Plant`,
- применять эффекты напрямую,
- обходить `Shelf`.

## Source Of Truth

Для текущего этапа source of truth такой:
- `Pot` — контейнер для `Plant`,
- `Pot` не является actor и не участвует в event loop как исполнитель,
- `Plant` живет внутри `Pot`, а не напрямую в `Slot`,
- влияние `Pot` является постоянным контейнерным слоем, а не timed modifier или instant effect,
- `Pot` не хранит runtime state `Plant` и не управляет его временем,
- совместимость и любые взаимодействия координируются через `Shelf`.
