# Правила Для Git

## Перед Любым Коммитом

Перед тем как делать коммит, агент обязан посмотреть этот файл и следовать ему.

## Основное Правило

Изменения нужно коммитить отдельными коммитами с отдельным смыслом.

Нельзя без необходимости смешивать в одном коммите:
- изменения логики,
- изменения UI,
- тесты,
- документацию,
- правила для агентов,
- правки ассетов.

Если изменения можно разумно разделить на несколько независимых частей, их нужно коммитить отдельно.

## Ожидания Для Коммита

Перед коммитом агент должен:
1. посмотреть `git status`,
2. понять, какие изменения относятся к каким задачам,
3. сгруппировать их по смыслу,
4. сделать отдельные коммиты для разных смысловых блоков,
5. не включать в коммит чужие или случайные несвязанные правки.

## Сообщения Коммитов

Сообщение коммита должно:
- быть коротким,
- описывать смысл изменения,
- не быть слишком общим вроде `fix stuff`,
- отражать только тот набор изменений, который реально попал в коммит.

## Проверка Перед Push

Перед `push` агент должен убедиться, что:
- изменения разбиты на логичные коммиты,
- в коммитах нет случайных файлов,
- если были тесты, понятно, какие из них запускались.
