# Контракт Для Targeting Rules

## Назначение

Этот документ фиксирует правила выбора целей в локальной симуляции `Shelf`.

Это строгий архитектурный контракт для агентов, а не гайд по реализации.

## Общая Модель Targeting

В системе выбора целей:
- actor, например `Totem`, не выбирает конкретные цели,
- actor задает только target rule,
- `Shelf` отвечает за поиск кандидатов,
- `Shelf` отвечает за фильтрацию,
- `Shelf` отвечает за выбор конкретных целей.

## Разделение: Rule vs Selection

`Target rule` и `selection` — это разные сущности.

`Target rule`:
- описывает, какие цели нужны,
- задается actor-источником.

`Selection`:
- выбирает конкретные цели,
- выполняется только `Shelf`.

Жесткое правило:
- actor не должен смешивать rule и selection в своей логике.

## Базовые Target Rules

На текущем этапе допустим ограниченный набор target rules:
- `self`
- `adjacent`
- `all`
- `random_valid`
- `by_type`
- `by_tag`

Правило расширения:
- этот список не должен автоматически расширяться,
- новые rules добавляются только осознанно и отдельно.

## Adjacent

`Adjacent` определяется только внутри одного ряда.

Правила:
- adjacency считается только внутри одного и того же ряда,
- соседство определяется по позиции в ряду,
- элементы из других рядов не считаются adjacent,
- `col` сам по себе не используется как отдельное правило соседства вне row-based логики.

## All

`All` означает:
- все валидные цели внутри текущей `Shelf`,
- без выхода за пределы локальной симуляции.

Правила:
- `all` не включает другие `Shelf`,
- `all` не выходит за пределы текущей `Shelf`.

## Random Valid

`Random valid` означает:
- случайный выбор выполняется только `Shelf`,
- actor не выполняет random самостоятельно,
- выбор идет только из валидных кандидатов,
- random должен быть централизованным.

## By Type / By Tag

`By type` и `by tag` означают:
- filtering по типу и тегам выполняется на стороне `Shelf`,
- actor задает условие,
- `Shelf` применяет его к кандидатам.

## Фильтрация Целей

После выбора кандидатов цель может отклонить эффект.

Правила:
- объект может игнорировать неприменимые modifiers и effects,
- это не отменяет сам selection,
- это влияет только на итоговое применение.

## Ограничения На Цели

Целями могут быть только actors:
- `Plant`,
- `Totem`.

Также:
- `Pot` не является стандартной целью,
- `Slot` не является целью,
- `Room` не является целью.

## Отсутствие Self-Targeting По Умолчанию

По умолчанию actor не должен выбирать себя как цель.

Правило:
- если self-targeting нужен, это должно быть явно задано правилом `self`.

## Timing Выбора Целей

Правила:
- target selection выполняется в текущем тике при построении следующей очереди,
- применение эффекта не происходит в момент selection,
- применение эффекта происходит только в следующем тике через `APPLY`.

Типовая схема:
- `N`: actor делает activation report,
- конец `N`: `Shelf` строит gameplay event для `N+1`,
- `N+1`: actor реагирует и создает `request`,
- конец `N+1`: `Shelf` выполняет selection,
- `N+2`: `Shelf` применяет эффект.

## Детерминизм

Targeting должен быть детерминированным при одинаковом состоянии.

Правила:
- порядок обхода `Shelf` идет как `row -> col`,
- этот порядок влияет на результат,
- одинаковое состояние дает одинаковый результат выбора,
- random централизован и контролируем со стороны `Shelf`.

## Targeting Не Должен

Targeting не должен:
- выполняться внутри actor,
- использовать сложные вложенные правила,
- зависеть от служебных данных event loop,
- выходить за пределы одной `Shelf`,
- использовать полную 2D-соседность между рядами,
- позволять неявный self-targeting,
- дублировать логику в нескольких местах.

## Source Of Truth

Для текущего этапа source of truth такой:
- actor задает только target rule,
- `Shelf` выполняет поиск кандидатов, filtering и конкретный selection,
- adjacency считается только внутри одного ряда,
- random target selection выполняется только `Shelf`,
- selection выполняется при построении следующей очереди,
- применение происходит только в следующем тике,
- стандартными целями считаются только `Plant` и `Totem`,
- targeting не выходит за пределы локальной симуляции одной `Shelf`.
