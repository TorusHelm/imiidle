# Контракт Для Room

## Назначение

Этот документ фиксирует runtime-поведение и архитектурную роль `Room`.

Это строгий контракт для агентов, а не гайд по реализации.

## Роль Room

`Room` — это верхнеуровневый контейнер.

`Room`:
- содержит слоты для размещения `Shelf`,
- управляет доступностью и структурой этих слотов,
- задает верхний уровень игрового пространства над `Shelf`,
- не заменяет `Shelf` как orchestrator локальной симуляции.

## Room Как Контейнер Со Слотами Под Shelf

Жесткие правила:
- `Room` содержит собственные слоты для `Shelf`,
- слот `Room` может быть пустым,
- в слот `Room` может быть помещен `Shelf`,
- количество доступных слотов в `Room` может расширяться через игровые механики.

Следствие:
- `Room` управляет размещением `Shelf` как структурой комнаты,
- `Room` не подменяет собой внутреннюю структуру `Shelf`.

## Связь Room И Shelf

`Room` может содержать несколько `Shelf`.

Правила:
- каждая `Shelf` остается отдельной локальной зоной симуляции,
- `Room` не объединяет несколько `Shelf` в одну общую event-систему,
- события одной `Shelf` не передаются в другую `Shelf`.

## Границы Ответственности

`Room` отвечает за:
- хранение и размещение `Shelf`,
- управление слотами под `Shelf`,
- верхнеуровневые правила комнаты,
- доступность и расширение пространства под `Shelf`.

`Room` не отвечает за:
- локальный tick `Shelf`,
- event loop внутри `Shelf`,
- маршрутизацию локальных gameplay events между actors,
- связывание разных `Shelf` между собой.

## Room Influence

`Room` может задавать глобальные пассивные правила внутри конкретной комнаты.

Эти правила:
- могут влиять на `Shelf`,
- могут влиять на `Plant` по фильтрам или условиям,
- действуют как внешний слой контекста симуляции.

Примеры уровня абстракции:
- условия комнаты,
- room-wide пассивные эффекты,
- фильтруемые глобальные правила для `Plant`.

## Модель Влияния Room

`Room influence`:
- не является timed modifier,
- не является instant effect,
- не является потоком событий,
- не должно спамить event loop,
- не использует event queue,
- учитывается как постоянный контекст во время `UPDATE`.

Архитектурное правило:
- `Room influence` — это отдельный постоянный верхнеуровневый слой влияния,
- `Shelf` учитывает его как часть контекста симуляции.

## Слои Влияния В Системе

Итоговое поведение объектов может зависеть от нескольких слоев:
- базовые параметры,
- timed modifiers,
- aura-влияния,
- влияние `Pot`,
- влияние `Room`.

Это архитектурный принцип, а не отдельная реализация.

## Отсутствие Cross-Shelf Взаимодействий

Жесткие правила:
- `Shelf` внутри одной `Room` не взаимодействуют друг с другом,
- actor из одной `Shelf` не может влиять на actor в другой `Shelf` напрямую,
- `Room` не маршрутизирует события между `Shelf`,
- cross-shelf реакции через event chain отсутствуют.

## Room Не Является Actor

`Room`:
- не является actor,
- не участвует в локальной симуляции как `Plant` или `Totem`,
- не имеет activation,
- не генерирует локальные gameplay events как actor,
- не заменяет event loop `Shelf`.

## Room Не Является Обычной Target-Сущностью

`Room`:
- не является стандартной целью для modifiers и instant effects,
- не участвует в targeting-системе как обычный gameplay target,
- существует как отдельный верхнеуровневый слой влияния, а не как цель эффектов.

## Room Как Gameplay-Сущность

В этом документе `Room` рассматривается как gameplay-контейнер верхнего уровня.

Правила:
- это не просто визуальная сцена,
- это не просто корневой узел Godot,
- `Room` описывает структуру игрового пространства над `Shelf`.

## Room Не Должен

`Room` не должен:
- подменять собой `Shelf`,
- становиться глобальным event bus,
- маршрутизировать локальные события между `Shelf`,
- напрямую изменять runtime state actors в обход `Shelf`,
- быть actor,
- генерировать локальные actor-события,
- связывать `Shelf` между собой,
- становиться обычной target-сущностью.

## Source Of Truth

Для текущего этапа source of truth такой:
- `Room` — верхнеуровневый контейнер для размещения `Shelf`,
- `Room` управляет только структурой и верхнеуровневым контекстом комнаты,
- каждая `Shelf` внутри `Room` остается отдельной локальной зоной симуляции,
- `Room influence` является постоянным внешним слоем, а не timed modifier, instant effect или event flow,
- `Room influence` не использует event queue и учитывается в `UPDATE`,
- `Room` не маршрутизирует события между `Shelf` и не превращает их в общую систему.
