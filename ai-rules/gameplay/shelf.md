# Контракт Для Shelf

## Назначение

Этот документ фиксирует runtime-поведение и архитектурную роль `Shelf`.

Это строгий контракт для агентов, а не гайд по реализации.

## Роль Shelf

`Shelf` — это локальный orchestrator симуляции.

`Shelf`:
- является контейнером `Slot`,
- является границей локальных gameplay-взаимодействий,
- координирует actor-взаимодействия только внутри своей полки,
- не является глобальным event bus,
- не связывает разные `Shelf` между собой.

Все actor-взаимодействия внутри `Shelf` проходят только через `Shelf`.

## Shelf Как Локальная Игровая Область

Каждая `Shelf` является отдельной локальной зоной симуляции.

Правила:
- взаимодействия `Plant` и `Totem` ограничены пределами одной `Shelf`,
- локальные события не выходят за пределы `Shelf`,
- `Shelf` не взаимодействует с другими `Shelf` напрямую.

## Структура Shelf

`Shelf` представляет собой сетку в формате `row` / `col`.

Жесткие правила:
- `Shelf` не ограничивается одной линейной полосой,
- позиция объекта в формате `row` / `col` имеет gameplay-значение,
- порядок размещения в `Shelf` влияет на результат,
- обработка и обход элементов происходят по рядам.

Уточнение:
- система учитывает позицию объектов через `row` / `col`,
- но логика обхода и обработки остается row-based.

## Adjacent Внутри Shelf

`Adjacency` считается только внутри одного и того же ряда.

Правила:
- соседство определяется по позиции в ряду,
- объекты из разных рядов не считаются adjacent,
- `row` / `col` влияет на targeting и gameplay-логику.

Это является частью осознанной механики расстановки.

## Shelf И Slot

`Shelf` содержит набор `Slot`.

Правила:
- `Slot` является тупым контейнером,
- `Slot` не участвует в симуляции как actor,
- `Shelf` знает структуру `Slot` и их порядок,
- пустой `Slot` является нормальным состоянием.

## Что Может Находиться В Slot

`Slot` внутри `Shelf` может содержать:
- `Pot`,
- `Totem`,
- или быть пустым.

Правила:
- `Plant` не помещается в `Slot` напрямую,
- `Plant` существует только внутри `Pot`,
- `Shelf` валидирует допустимое содержимое `Slot`.

## Размещение И Управление Содержимым

`Shelf` отвечает за:
- размещение содержимого внутри своих `Slot`,
- перемещение содержимого внутри своих `Slot`,
- удаление содержимого из своих `Slot`,
- проверку допустимости размещения,
- знание того, какие `Slot` заняты, а какие пусты.

`Slot` сам не принимает решений о размещении.

## Shelf И Event Loop

`Shelf`:
- владеет локальным tick,
- владеет локальной очередью событий,
- управляет deferred-обработкой событий.

Важно:
- event loop является частью runtime `Shelf`,
- подробные фазы работы описываются отдельно в `ai-rules/gameplay/shelf-event-loop.md`.

## Shelf И Routing Событий

`Shelf`:
- принимает `report` и `request` от локальных actors,
- маршрутизирует локальные gameplay events,
- применяет эффекты к целям только в рамках своей локальной симуляции,
- является единственной точкой координации actor-взаимодействий внутри полки.

## Shelf И Выбор Целей

Конкретный выбор целей выполняет `Shelf`.

Actor, например `Totem`, может задавать только target rule.

`Shelf` выбирает валидные цели на основе:
- структуры `Shelf`,
- позиции `row` / `col`,
- соседства внутри ряда,
- фильтров по типу, тегам и условиям,
- правил random target selection.

Жесткое правило:
- random target selection выполняется только `Shelf`.

## Shelf Как Интегратор Локального Контекста

`Shelf` учитывает локальные и внешние слои влияния при симуляции.

В рамках своей локальной симуляции `Shelf` должен учитывать:
- timed modifiers целей,
- aura-влияния `Totem`,
- влияние `Pot`,
- влияние `Room`.

`Shelf` является точкой, где эти слои сходятся в локальный gameplay context.

## Shelf Не Владеет Runtime Actors

Жесткие правила:
- `Shelf` не является source of truth для runtime state `Plant` и `Totem`,
- `Shelf` не хранит их `progress`, `age`, `count` и modifiers как основной runtime,
- runtime-состояние живет в самих сущностях,
- `Shelf` только координирует и учитывает их в симуляции.

## Детерминизм И Порядок

Правила:
- порядок `Slot` в `Shelf` имеет gameplay-значение,
- `row` / `col` и порядок обхода влияют на результат,
- `Shelf` должен вести себя детерминированно при одинаковом состоянии и одинаковых входных событиях,
- порядок обработки считается частью системы, а не побочным эффектом.

## Shelf Не Должен

`Shelf` не должен:
- становиться глобальным event bus,
- взаимодействовать с другими `Shelf` напрямую,
- хранить runtime actors как source of truth,
- делегировать orchestration в `Slot`,
- позволять actor -> actor взаимодействия в обход `Shelf`,
- трактовать adjacency как полную 2D-соседность между рядами,
- применять эффекты вне своей локальной симуляции,
- ломать deferred-модель событий.

## Source Of Truth

Для текущего этапа source of truth такой:
- `Shelf` — локальный orchestrator и локальная игровая область,
- `Shelf` содержит и упорядочивает `Slot` в сетке `row` / `col`,
- все actor-взаимодействия внутри полки проходят только через `Shelf`,
- adjacency считается только внутри одного ряда,
- `Shelf` владеет локальным tick, event loop и target selection,
- `Shelf` не владеет runtime state actors как основным источником данных,
- `Shelf` не взаимодействует с другими `Shelf` и не выводит локальные события за пределы своей полки.
